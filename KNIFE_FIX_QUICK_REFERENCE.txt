💡 WHY THIS WORKS
═════════════════════════════════════════════════════════════════════════════

External tools can't hook functions, so they use:

1. State Invalidation
   → Tells game "item is new"
   → Forces re-initialization

2. Multiple Writes
   → Increases chance one sticks
   → Redundancy for reliability

3. Longer Delays
   → Gives game time to process
   → Accounts for system speed variation

4. Dual Regeneration
   → First: triggers reload
   → Second: ensures completion
   → CRITICAL for external tools!

5. Verification
   → Confirms change actually worked
   → Helps with debugging

🎓 TECHNICAL NOTES
═════════════════════════════════════════════════════════════════════════════

Why stage 9 is critical:
→ Many external tools only do 1 regeneration
→ Game needs time to fully load assets
→ Second call ensures completion
→ This is the difference between 40% and 80%

Why delays matter:
→ External writes are transient
→ Game takes time to process changes
→ More delay = more reliable
→ Trade-off: speed vs reliability

Why verification is important:
→ Proves change actually applied
→ Helps debug if something fails
→ Shows ✓ in console on success
